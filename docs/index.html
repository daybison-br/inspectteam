04 · dados e segurança

Multi-tenancy e persistência

O banco é compartilhado entre tenants, com tenant_id nas entidades operacionais e políticas de isolamento aplicadas pelo PostgreSQL.

Autenticação

  • JWT HMAC SHA-256
  • Access token: 15 minutos
  • Refresh token: 30 dias, armazenado como hash
  • BCrypt com custo 12
  • API stateless

Autorização

  • Owner com controle do tenant
  • Administrador global identificado no token
  • Papéis customizados por tenant
  • Grants por formulário
  • Política padrão: negar

06 · formulários

Definição e versionamento

A definição é armazenada como JSONB. IDs de campos devem ser estáveis e únicos para preservar respostas entre sincronizações.

texttextareanumberdatetimeselectmultiselectcheckboxphotosignatureheadinginstructions

    }]

Draft

Editável por usuários com FORM_EDIT. A definição é validada antes de persistir.

Publicado

Imutável e utilizado pelas submissões. A publicação cria automaticamente o próximo draft.

07 · offline e arquivos

Sincronização preparada para mobile

O protocolo inicial permite que o futuro aplicativo mantenha uma fila local e reenvie operações com segurança após recuperar conexão.

RegistrardeviceId vinculado à membership
Pullversões autorizadas e tombstones
Editar offlineUUIDs gerados no cliente
PushCREATE, UPDATE e COMPLETE
ConfirmarmutationId impede duplicação

Concorrência

A propriedade revision aplica controle otimista. Revisão divergente deve gerar conflito e não sobrescrever silenciosamente.

Arquivos

Fotos e assinaturas usam upload direto ao MinIO por URL pré-assinada. O limite atual é 20 MB e a API confirma o tamanho esperado.

!
Ainda é um protocolo inicial

Banco local mobile, fila de uploads, política avançada de conflitos e experiência de recuperação permanecem como trabalho futuro.

Configurar Nginx

Salve em /etc/nginx/sites-available/inspecteam. O proxy da API no mesmo host evita a necessidade de CORS para o futuro painel web.

Nginx
server {
    listen 80;
    listen [::]:80;
    server_name app.seudominio.com;

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

server {
    listen 80;
    listen [::]:80;
    server_name storage.seudominio.com;
    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:9000;
        proxy_http_version 1.1;
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_request_buffering off;
    }
}

Migrações podem impedir rollback

Antes de publicar, confirme que a versão anterior da aplicação continua compatível com as novas migrações. Prefira alterações de banco expansíveis e reversíveis.

Backup do PostgreSQL

Backup lógico
mkdir -p /opt/inspecteam/backups
cd /opt/inspecteam/shared
docker compose -f compose.production.yaml exec -T postgres \
  pg_dump -U inspecteam_owner -d inspecteam -Fc \
  > /opt/inspecteam/backups/postgres-$(date +%F-%H%M).dump

Restaure primeiro em ambiente separado:

Teste de restauração
cat /opt/inspecteam/backups/<ARQUIVO>.dump | \
docker compose -f compose.production.yaml exec -T postgres \
  pg_restore -U inspecteam_owner -d <BANCO_DE_TESTE> --clean --if-exists

Backup dos objetos MinIO

Use snapshots do volume da VPS ou uma ferramenta compatível com S3 para espelhar o bucket em armazenamento externo. Para uma cópia local consistente, pare brevemente o MinIO e compacte o volume correto.

Janela de manutenção
cd /opt/inspecteam/shared
docker compose -f compose.production.yaml stop minio
docker volume ls
docker run --rm \
  -v <NOME_DO_VOLUME_MINIO>:/data:ro \
  -v /opt/inspecteam/backups:/backup \
  alpine tar czf /backup/minio-$(date +%F-%H%M).tgz -C /data .
docker compose -f compose.production.yaml start minio
  • Mantenha cópias fora da VPS.
  • Criptografe backups que contenham dados pessoais.
  • Defina retenção e teste restauração periodicamente.
  • Monitore espaço em disco, RAM, certificados e falhas de login.

Aplicação web e administração global

InspecTeam Design System

A interface utiliza tokens semânticos, temas claro e escuro, iconografia Fluent, componentes corporativos, navegação responsiva e padrões acessíveis inspirados em Material Design 3, Fluent 2 e Carbon.

Área do tenant

  • Cadastro e login reais.
  • Dashboard, formulários, respostas, equipe, papéis e configurações.
  • Editor persistido com 12 tipos de campo, seções, pré-visualização e publicação.
  • Menus e ações filtrados pelas permissões efetivas.
  • Tela de auditoria do tenant, tema claro/escuro e confirmações acessíveis.

Administração global

  • Métricas, busca e estado de tenants e usuários.
  • Promoção de administradores com proteção do último admin ativo.
  • Entrada auditada em tenant usando membership técnica.
  • Eventos elevados em platform_audit_events.

UPDATE users

WHERE LOWER(email) = LOWER('admin@seudominio.com');
